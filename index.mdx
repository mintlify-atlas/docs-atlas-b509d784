Zeytin is a lightweight 2D game engine written in C++, built on top of [Raylib](https://github.com/raysan5/raylib). It ships with a fully integrated editor powered by [Dear ImGui](https://github.com/ocornut/imgui), giving you a streamlined WYSIWYG workflow — from designing entities to tweaking properties live at runtime.

## What is Zeytin?

Zeytin separates the **engine** (your game logic and rendering) and the **editor** (a Dear ImGui desktop app) into two distinct executables that communicate over ZeroMQ. Your game code compiles directly into the engine — there is no scripting layer.

The core concept is **variants** (components). Every game object property you expose with the `PROPERTY()` macro becomes live-editable in the editor at runtime.

<Steps>
  <Step title="Clone and build">
    Clone the repository, then build the editor and engine using Premake5 and `make`.
  </Step>
  <Step title="Create variants">
    Define C++ classes that extend `Component` and register properties with `VARIANT()` and `PROPERTY()` macros.
  </Step>
  <Step title="Compose entities in the editor">
    Use the Hierarchy panel to create entities and add your variants. The editor auto-detects dependencies via the Query API.
  </Step>
  <Step title="Play and iterate">
    Click **Start Engine** to build and run. Tweak property values live — no recompile needed for data changes.
  </Step>
</Steps>
